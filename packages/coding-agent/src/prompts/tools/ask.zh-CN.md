在任务执行过程中，当你需要澄清或用户输入时，向用户提问。

<conditions>
- 存在多种方案，且它们之间有显著不同的权衡，用户应当进行权衡
</conditions>

<instruction>
- 使用 `recommended: <index>` 标记默认项（从 0 开始计数），自动添加“（推荐）”
- 对于多个相关问题，使用 `questions`，而不是一次只问一个
- 在问题上设置 `multi: true` 以允许多选
- 使用简短的选项标签；将解释性的权衡放在 `description` 中，而不是将其合并到标签里
</instruction>

<caution>
- 提供 2-5 个简洁且彼此不同的选项
</caution>

<critical>
- **默认采取行动。** 使用仓库约定、现有模式和合理默认值自行解决歧义后继续。提问前，先穷尽现有来源（代码、配置、文档、历史记录）。只有当不同选项存在用户必须决定的实质性权衡时，才提问。
- **如果多个选择都可以接受**，选择最保守／最标准的选项并继续；说明该选择。
- **不要包含“Other”选项**——UI 会自动为每个问题添加“Other（自行填写）”。
</critical>
