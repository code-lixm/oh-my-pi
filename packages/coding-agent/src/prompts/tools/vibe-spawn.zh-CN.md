启动一个持久化的 worker 会话——一个完整的编码代理（编辑、bash、grep、所有功能），由你通过对话来驱动。根据任务选择 CLI 版本：

- `fast`：用于机械性、定义明确工作的低延迟模型（重命名、样板代码、运行测试、数据收集）。
- `good`：用于高难度工作的强大模型（设计、调试、多文件修改、判断性决策）。

`prompt` 是该会话的第一条指令。worker 在此之外启动时没有任何上下文——请包含文件、约束条件和验收标准。`name`（可选）为会话添加标签，否则将自动生成一个。

立即返回会话 id；该轮结果（活动轨迹 + worker 的响应）会在 worker 完成时自动发送给你。除非你被阻塞，否则不要等待——继续指导其他会话。

该会话在本轮之后仍会持续存在：它会记住整个对话。使用 `vibe_send` 继续该会话；对于同一工作流中的后续事项，绝不要再启动第二个会话。
