向你的某个 worker 会话发送消息（使用来自 `vibe_spawn` / `vibe_list` 的 id）。该会话会保留其完整的对话历史——可以自然地引用之前的工作（“现在对另一个模块也做同样的事”）。

立即返回一个 ack，告诉你消息是如何送达的：

- `turn` — worker 当时空闲；已开始一个新轮次。完成后其结果会自行送达。
- `steered` — worker 当时正在处理中；你的消息已作为实时引导注入到正在运行的轮次中。
- `queued` — worker 当时正在处理中且当前不可引导；你的消息会自动作为下一轮运行。

将其用于后续跟进、更正、范围变更和审查请求。不要重新解释之前的上下文——该会话已经有了。
