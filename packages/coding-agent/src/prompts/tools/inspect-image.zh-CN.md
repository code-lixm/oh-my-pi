使用具备视觉能力的模型检查图像文件，并返回紧凑的文本分析。

<instruction>
- 将此用于图像理解任务（OCR、UI/截图调试、场景/对象问题）
- 提供 `path` 作为本地图像文件路径、`Image #N` 附件标签或 `attachment://N` URI
- 编写具体的 `question`：
  - 要检查的内容
  - 约束（例如："逐字引用可见文本"、"仅报告已确认的发现"）
  - 期望的输出格式（项目符号/表格/JSON/简短回答）
- 保持 `question` 基于可观察证据，并在细节不清楚时要求说明不确定性
- 当目标是图像分析时，使用此工具而不是 `read`
</instruction>

<output>
- 返回来自视觉模型的纯文本分析
- 工具输出中不会返回任何图像内容块
</output>

<critical>
- 如果设置阻止提交图像，该工具将以可操作的错误失败
- 如果已配置的模型不支持图像输入，请在重试前配置具备视觉能力的模型角色
</critical>
