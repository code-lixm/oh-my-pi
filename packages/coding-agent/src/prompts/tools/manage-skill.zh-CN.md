创建、更新或删除托管技能——一种写入隔离目录（`~/.omp/agent/managed-skills`）并在后续会话中像普通技能一样呈现的`SKILL.md`。

托管技能用于值得编码固化的可重复流程：一套设置顺序、一个调试方案、一个项目特定的工作流。它们与用户编写的技能分开保存，而这个工具 NEVER 会编辑那些技能。

- `action: "create"`——如果该技能已存在则失败。
- `action: "update"`——覆盖正文；如果该技能不存在则失败。
- `action: "delete"`——如果该技能不存在则失败。

`name` 使用短横线命名法（小写字母、数字、连字符）。`description` 用于驱动发现，因此请写得具体。不要在 `body` 中包含前置元数据；它会根据 `name` 和 `description` 生成。
