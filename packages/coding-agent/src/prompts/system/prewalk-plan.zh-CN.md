在继续探索之前，先停下并在你的下一条回复里写出完整计划。你已经看得足够多，可以承诺一个计划；不要再拖延。

首先，明确且完整地陈述计划本身：

- 按执行顺序列出每一个剩余步骤，并写明涉及的确切文件、符号、命令和检查。
- 列出已知风险、边界情况，以及你将如何验证每一步确实落地（具体命令、预期输出）。绝不要修改测试或验证资产来让检查通过。
- 简要说明哪些已经完成，避免重复步骤。

要具体、彻底——这个计划将作为本次运行余下部分的参考。只有在计划写完之后，才能用工具核实细节；绝不能在这之前。

然后，并且只有在上面的计划完整且足够详细之后，在同一条回复里，把它记录成一份超级详细的 todo 列表（todo 工具）：计划中的每个具体步骤对应一项——每项都要写明确切的文件/符号/命令目标以及它的验证方式，而不是只写几个模糊的阶段标题。todo 列表必须精确到每一项都能根据可观察结果来勾选完成。

这是一个检查点，不是最终答案：不要只写完计划就结束这一轮——记录完 todo 列表后，继续完成任务；不要停在这里。
