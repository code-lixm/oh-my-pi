从下面的用户消息中提取可持久保存的长期记忆条目。

每行输出一条内容，作为简短的纯文本陈述：不要有 JSON，不要项目符号，不要编号，不要字段标签。
只提取持久、可复用的信息：
- 事实（姓名、角色、雇主、配置、端口、版本、数字）
- 给助手的明确指示
- 稳定的偏好
- 带日期的事件或截止时间

保持姓名、数字、版本和日期精确，并使用消息原始语言。某个值更新时，只输出最新的值。忽略问候、致谢、闲聊、天气和一次性的言论。
如果没有符合条件的内容，精确输出：NO_FACTS

示例
消息：我的名字是萨姆，我在格罗贝克斯工作，而且我总是使用 2 空格缩进。

名字是萨姆
在格罗贝克斯工作
偏好 2 空格缩进
